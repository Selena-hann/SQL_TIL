## 1-1. BigQuery 기본지식

~~~
✅ 학습 목표 :
* 데이터 관련 기초 지식(OLTP, SQL, Row, Column, 저장 형태 등)을 설명할 수 있다. 
* BigQuery 관련 기초 지식에 대해서 파악할 수 있다. 
~~~

### 데이터는 보통 데이터베이스 테이블 등에 저장한다.
- Database: 데이터의 저장소
- Table: 데이터가 저장된 공간
- 저장된 데이터를 제품(앱,웹)에서 사용

- 예시: 음식 배달 서비스의 데이터 - 유저, 음식점, 음식, 라이더 등

- OLTP: 거래를 하기 위해 사용되는 데이터베이스 (보류, 중간 상태가 없음.)

### BigQuery 장점
- SQL 이용해 쉽게 데이터 추출 가능
- OLAP 도구이므로 속도가 빠름
- Firebase, Google Analytics4의 데이터를 쉽게 추출 가능 (사용기기, 위치, OS버전, 이벤트 행동 획득 가능(별도의 로깅 필요))
- 데이터 웨어하우스를 사용하기 위해 서버를 띄울 필요 없음. 구글에서 인프라를 관리함.


## 2-1. 데이터 활용 Overview

~~~
✅ 학습 목표 :
* 데이터를 활용하는 과정을 설명할 수 있다.
* 데이터를 탐색하는 과정으로 조건과 추출, 요약을 할 수 있다. 
~~~

~~~
어떤 과제를 맡았을 때 혹은 일을 할 때, 어떤 일을 해야한다 > 원하는 것을 정한다 의 단계에서는 문제 정의가 중요하다. (무엇을 어떻게 할 것인지, 타겟팅은 누구인지 등)
지표 정의를 한 후 가설을 만들고,
그 후 데이터 탐색을 하는데 단일 자료, 다량의 자료로 나뉜다.
여기서 필요한 것이 데이터 조건(필터링), 추출, 변환, 요약의 과정이다.
이후 데이터 결과를 검증하고
피드백/활용한다.
~~~


## 2-2. 저장된 데이터 활용하기

~~~
✅ 학습 목표 :
* 데이터가 저장되는 형태를 알고 저장된 데이터를 활용할 수 있다. 
~~~

### ERD가 없다면
- 어떤 테이블, 칼럼 존재?
- 다른 테이블과 연결할 때 어떤 칼럼 사용?
- 칼럼의 값들은 어떤 의미를 가지는가?

### 서비스에 사용될 데이터베이스
- 유저 테이블
- 배송 테이블
- 물건 테이블

### 앱/웹 로그 데이터
- 유저가 앱/웹에 들어와서 회원 가입 - 페이지 확인, 컨텐츠 확인 등등의 데이터
- "과정"을 알 수 있는 데이터

### 공공 데이터, 서드파티 데이터
- 날씨
- 페이스북 광고 데이터

~~~
